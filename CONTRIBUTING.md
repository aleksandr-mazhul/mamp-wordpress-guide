# Участие в гайде

Спасибо, что хотите улучшить документацию. Репозиторий — только markdown и скриншоты, без кода приложения.

---

## Структура

Весь контент — в `docs/`:

```
docs/local/     — Часть 1 (Mac + MAMP)
docs/migrate/   — Часть 2 (перенос)
docs/hosting/   — Часть 3 (в разработке)
```

Скриншоты — в `assets/images/<часть>/`. См. [assets/images/README.md](assets/images/README.md).

---

## Шаблон шагового файла

```markdown
# NN. Заголовок

[навигация]

---

## Сделайте

1. Действие
2. Действие

**Проверка:** что должно получиться.

---

## Пояснение

<details>
<summary>Зачем это нужно</summary>
Короткий текст.
</details>

---

## Если ошибка

| Симптом | Куда |
|---------|------|
| ... | [local/troubleshooting.md#anchor-id](../docs/local/troubleshooting.md#anchor-id) |
```

Правила:

- **Сделайте** — только императив, без альтернативных путей в основном тексте
- Альтернативы (FTP, плагин) — в `appendix-*.md`
- Таблицы параметров — в README части, не дублировать в каждом шаге
- Без emoji в заголовках

---

## Скриншоты

| Правило | Значение |
|---------|----------|
| Формат | PNG |
| Ширина | до 1200 px |
| Имя файла | kebab-case, латиница (`mamp-ports-settings.png`) |
| Путь | `assets/images/local/` или `assets/images/migrate/` |
| В markdown | относительный путь от файла документа |

После добавления файла обновите таблицу в [assets/images/README.md](assets/images/README.md).

---

## Якоря в troubleshooting

Используйте явные HTML-якоря с латинскими id:

```markdown
<a id="port-80"></a>

## Port 80 занят
```

Ссылки из шагов: `troubleshooting.md#port-80`

---

## Проверка перед PR

1. Относительные ссылки ведут на существующие файлы
2. Новые скриншоты добавлены в `assets/images/README.md`
3. Русский язык, простые формулировки
