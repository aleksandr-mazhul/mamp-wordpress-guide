## Сделайте

### Создание базы в панели хостинга

1. Панель хостинга → **MySQL Databases** / **Базы данных**
2. Блок **Create Database** / **Создать базу**:
   - введите имя латиницей, например `my_wp_db`
   - нажмите **Create** / **Создать**
   - **скопируйте полное имя** базы (хостинг может добавить префикс, например `user_my_wp_db`)
3. Блок **Create User** / **Пользователь MySQL**:
   - имя пользователя, например `my_wp_user`
   - надёжный пароль → **Create**
   - **запишите** логин и пароль
4. Блок **Add User to Database** / **Привязать пользователя**:
   - выберите пользователя и базу
   - привилегии: **ALL PRIVILEGES** / **Все права** → **Save**

### Запишите 4 поля для wp-config

5. В том же разделе MySQL найдите **Database details** / **Сведения о БД** и заполните:

| Поле | Ваше значение | Куда попадёт |
|------|----------------|--------------|
| **DB_NAME** | полное имя базы | `wp-config.php` |
| **DB_USER** | пользователь БД | `wp-config.php` |
| **DB_PASSWORD** | пароль пользователя | `wp-config.php` |
| **DB_HOST** | хост из панели | `wp-config.php` |

> **DB_HOST** — скопируйте из панели. Не ставьте `localhost` по привычке с MAMP, если панель показывает другое (например `sql123.hosting.com`).

### Импорт SQL через phpMyAdmin

6. Панель → **phpMyAdmin** (новая вкладка браузера)
7. Слева нажмите на **пустую** базу, которую только что создали
8. Вкладка **Импорт** (Import)
9. **Выберите файл** → укажите `.sql` с Mac ([шаг 1](01-prepare.md))
10. Формат: **SQL** (обычно определяется сам)
11. Нажмите **Вперёд** (Go) внизу страницы
12. Дождитесь сообщения об успехе — слева появятся таблицы `wp_*`

## Пояснение

<details>
<summary>Импорт занимает 1–2 минуты</summary>

Для учебного сайта это нормально. Не закрывайте вкладку, пока не увидите успех.
</details>

<details>
<summary>Файл слишком большой</summary>

Сожмите `.sql` в `.zip` и импортируйте zip, или см. [troubleshooting.md#sql-too-large](troubleshooting.md#sql-too-large).
</details>

## Если ошибка

| Симптом | Куда |
|---------|------|
| SQL слишком большой | [troubleshooting.md#sql-too-large](troubleshooting.md#sql-too-large) |
| Импорт в не ту базу | Выберите **пустую** базу хостинга, не старую с данными |
| Ошибка доступа в phpMyAdmin | Логин/пароль — пользователь БД из п. 3–4 |
