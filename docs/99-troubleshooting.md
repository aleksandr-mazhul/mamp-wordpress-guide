# 99. Решение проблем

[← Назад к оглавлению](../README.md)

Сборник типичных ошибок при работе с MAMP и WordPress. Перед поиском решения убедитесь, что MAMP запущен (Apache и MySQL — зелёные индикаторы).

---

## Порт 80 занят

**Симптом:** MAMP не запускает Apache и сообщает, что порт `80` занят.

**Причина:** Другая программа уже использует порт 80 — часто встроенный Apache macOS, Docker или другой веб-сервер.

**Решение:**

1. Проверьте, кто занимает порт:

   ```bash
   sudo lsof -i :80
   ```

2. Если это встроенный Apache macOS — остановите его:

   ```bash
   sudo apachectl stop
   ```

3. Перезапустите MAMP (**Stop** → **Start**)

**Альтернатива:** если освободить порт 80 не получается, верните стандартные порты MAMP в **Preferences → Ports** (`8888` для Apache, `8889` для MySQL). Тогда URL сайта будет `http://localhost:8888/название-вашей-папки/`, а в WordPress укажите сервер БД `localhost:8889`.

---

## Порт 3306 занят

**Симптом:** MySQL в MAMP не стартует, порт `3306` занят.

**Причина:** На Mac уже запущен другой MySQL (Homebrew, Docker, отдельная установка).

**Решение:**

1. Проверьте:

   ```bash
   sudo lsof -i :3306
   ```

2. Остановите конфликтующий MySQL или смените порт в MAMP (**Preferences → Ports** → MySQL Port: `8889`)

3. Если сменили порт MySQL — в WordPress укажите сервер БД `localhost:8889` вместо `localhost`

---

## Error establishing a database connection

**Симптом:** WordPress показывает «Ошибка установки соединения с базой данных».

**Частые причины и решения:**

| Причина | Решение |
|---------|---------|
| MySQL не запущен | MAMP → **Start**, дождитесь зелёного индикатора MySQL |
| Неверное имя БД | Проверьте в phpMyAdmin, что база существует |
| Неверный логин/пароль | Используйте `root` / `root` |
| Неверный host | Укажите `localhost` (при порте 3306) |
| Нестандартный порт MySQL | Если MySQL на `8889` — укажите `localhost:8889` |

**Проверка wp-config.php:**

Откройте `/Applications/MAMP/htdocs/название-вашей-папки/wp-config.php`:

```php
define( 'DB_NAME', 'wordpress' );      // ваше имя базы
define( 'DB_USER', 'root' );
define( 'DB_PASSWORD', 'root' );
define( 'DB_HOST', 'localhost' );       // или localhost:8889
```

## Страница не найдена (404)

**Симптом:** Браузер показывает 404 при открытии `http://localhost/название-вашей-папки/`.

**Решение:**

1. Проверьте, что папка лежит в `/Applications/MAMP/htdocs/`
2. Убедитесь, что внутри есть `index.php`
3. Имя папки в URL должно **точно совпадать** с именем в `htdocs` (регистр важен)
4. Apache должен быть запущен (зелёный индикатор)

---

## Белый экран

**Симптом:** Страница полностью белая, без текста.

**Решение:**

1. Включите отладку в `wp-config.php`:

   ```php
   define( 'WP_DEBUG', true );
   define( 'WP_DEBUG_DISPLAY', true );
   ```

2. Обновите страницу — должна появиться конкретная ошибка
3. Частая причина — несовместимый плагин: удалите папку плагина из `wp-content/plugins/`
4. Проверьте лог: `/Applications/MAMP/logs/php_error.log`

---

## Apache не стартует

**Симптом:** Индикатор Apache в MAMP остаётся красным.

**Решение:**

1. Проверьте, не занят ли порт 80 (см. выше)
2. Закройте Docker и другие веб-серверы
3. Перезапустите Mac
4. Посмотрите лог: `/Applications/MAMP/logs/apache_error.log`

---

## phpMyAdmin не открывается

**Симптом:** `http://localhost/phpMyAdmin/` не загружается.

**Решение:**

1. Убедитесь, что Apache **и** MySQL запущены
2. Попробуйте `http://127.0.0.1/phpMyAdmin/`
3. Откройте через WebStart: `http://localhost/MAMP/` → MySQL → phpMyAdmin
4. Проверьте порт Apache в **Preferences → Ports** (должен быть `80`)

---

## Сайт без стилей (голый HTML)

**Симптом:** Страница загружается, но без CSS.

**Решение:**

1. Проверьте URL — должен быть `http://localhost/название-вашей-папки/`
2. В админке: **Настройки → Общие** — проверьте «Адрес WordPress» и «Адрес сайта»
3. Оба должны начинаться с `http://localhost/название-вашей-папки`

---

## Чеклист

- [ ] MAMP запущен, Apache и MySQL — зелёные
- [ ] Порт Apache: `80`, MySQL: `3306`
- [ ] База данных создана в phpMyAdmin
- [ ] Файлы WordPress в `/Applications/MAMP/htdocs/название-вашей-папки/`
- [ ] В wp-config: `DB_HOST` = `localhost`, логин `root`, пароль `root`
- [ ] URL в браузере совпадает с именем папки
