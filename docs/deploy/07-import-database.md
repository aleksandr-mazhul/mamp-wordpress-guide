# 07. Импорт базы данных на хостинге

[← База на хостинге](06-create-remote-database.md) | [Оглавление](README.md) | [Далее: wp-config и URL →](08-wp-config-and-urls.md)

Загрузим файл `.sql` с Mac в базу на хостинге.

---

## Шаг 1. Открыть phpMyAdmin на хостинге

1. Войдите в **панель хостинга**
2. Найдите кнопку **phpMyAdmin** (часто рядом с MySQL Databases)
3. Откроется phpMyAdmin в новой вкладке

> Может попросить логин — используйте пользователя БД из [раздела 06](06-create-remote-database.md).

---

## Шаг 2. Выбрать базу

В левой панели нажмите на **пустую** базу, которую создали на хостинге (не путать с локальным именем).

---

## Шаг 3. Импорт

1. Вкладка **Импорт** (Import)
2. **Выберите файл** — укажите `.sql` с Mac (из [раздела 04](04-export-database.md))
3. Формат: SQL (обычно определяется автоматически)
4. Нажмите **Вперёд** (Go)

## Шаг 4. Проверить результат

Должно появиться сообщение об успешном импорте. Слева в базе появятся таблицы с префиксом `wp_` (или вашим префиксом).

---

## Если файл слишком большой

Симптом: ошибка про размер файла / `max upload size`.

| Решение | Описание |
|---------|----------|
| Сжать SQL в `.zip` | phpMyAdmin иногда принимает zip |
| Увеличить лимит | В панели хостинга или обратиться в поддержку |
| Разбить файл | Продвинутый вариант — для учебного сайта обычно не нужно |

Подробнее: [99-troubleshooting.md](99-troubleshooting.md).

---

## Не паникуйте, если…

Импорт занял 1–2 минуты — это нормально для среднего сайта.
