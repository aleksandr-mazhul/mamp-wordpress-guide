# 04. Экспорт базы данных с Mac

[← Бэкап](03-backup-local.md) | [Оглавление](README.md) | [Далее: Загрузка файлов →](05-upload-files.md)

База данных хранит статьи, настройки, пользователей. Сейчас сохраним её в файл `.sql` и перенесём на хостинг.

---

## Шаг 1. Запустить MAMP

1. Откройте **MAMP**
2. Нажмите **Start**
3. Дождитесь зелёных индикаторов Apache и MySQL

---

## Шаг 2. Открыть phpMyAdmin

В браузере:

```
http://localhost/phpMyAdmin/
```

Или: WebStart `http://localhost/MAMP/` → MySQL → phpMyAdmin.

---

## Шаг 3. Выбрать базу данных

В **левой панели** нажмите на имя вашей базы (ту, что создавали в [Части 1](../04-create-database.md), например `wordpress`).

---

## Шаг 4. Экспорт

1. Вверху откройте вкладку **Экспорт** (Export)
2. Метод: **Быстрый** (Quick)
3. Формат: **SQL**
4. Нажмите **Вперёд** (Go)

Браузер скачает файл, например `wordpress.sql`. Сохраните его в надёжное место (Рабочий стол или папка бэкапа).

---

## Почему нужен SQL-файл

| В файлах сайта | В базе данных |
|----------------|---------------|
| Код, темы, плагины | Тексты страниц и записей |
| Картинки в `wp-content/uploads` | Настройки, меню |
| `wp-config.php` | Логины пользователей |

Без базы на хостинге будет «пустой» WordPress без вашего контента.

---

## Частая ошибка

**Экспортировали не ту базу** — проверьте имя слева в phpMyAdmin. Оно должно совпадать с тем, что в `wp-config.php` (`DB_NAME`).
