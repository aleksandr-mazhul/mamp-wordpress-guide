# 03. Бэкап локального сайта

[← Выбор хостинга](02-choose-hosting.md) | [Оглавление](README.md) | [Далее: Экспорт БД →](04-export-database.md)

Перед переносом сделайте **резервную копию**. Если что-то пойдёт не так на хостинге, локальный сайт останется целым.

---

## Шаг 1. Скопировать папку сайта

1. Откройте Finder → **Переход** → **Переход к папке…** (⌘⇧G)
2. Введите:

   ```
   /Applications/MAMP/htdocs/
   ```

3. Найдите папку вашего сайта
4. Скопируйте её на **Рабочий стол** (⌘C → вставить на Рабочий стол)

Имя копии может быть, например: `backup-мой-сайт-2026`.

---

## Шаг 2. Записать локальный URL

Запишите точный адрес сайта — он понадобится при замене URL на хостинге:

```
http://localhost/название-вашей-папки/
```

Пример: `http://localhost/my-blog/`

> **Важно:** копируйте **без** слэша в конце или **со** слэшем — потом используйте тот же вариант в плагине замены (раздел 08).

---

## Шаг 3. Постоянные ссылки (ЧПУ)

Если ещё не делали в [Части 1](../06-first-launch.md):

1. Откройте локальную админку: `http://localhost/название-вашей-папки/wp-admin/`
2. **Настройки** → **Постоянные ссылки**
3. Выберите **Название записи**
4. **Сохранить изменения**

Это нужно, чтобы на хостинге работали красивые адреса страниц.

---

## Шаг 4. Экспорт базы (превью)

Полный экспорт — в [следующем разделе](04-export-database.md). Здесь достаточно понимать: вместе с папкой на Рабочем столе у вас будет и файл `.sql`.
