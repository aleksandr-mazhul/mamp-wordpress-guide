## Часть A. Настроить wp-config.php

### Шаг 1. Открыть файл на сервере

1. **Файловый менеджер** хостинга → корень сайта (`public_html`)
2. Найдите `wp-config.php`
3. **Edit** / **Редактировать**

### Шаг 2. Заменить данные базы данных

Найдите строки (значения с Mac **замените** на данные из [раздела 06](06-create-remote-database.md)):

```php
define( 'DB_NAME', 'имя_базы_на_хостинге' );
define( 'DB_USER', 'пользователь_базы' );
define( 'DB_PASSWORD', 'пароль_базы' );
define( 'DB_HOST', 'значение_из_панели_хостинга' );
```

Сохраните файл.

> **DB_HOST** — копируйте из панели хостинга, не из памяти про MAMP.

### Шаг 3. Проверить

Откройте в браузере адрес сайта (`http://ваш-домен...`).

- Если видите сайт, но ссылки ведут на `localhost` — переходите к **Части B**
- Если «Ошибка соединения с базой данных» — вернитесь к DB_NAME, DB_USER, DB_PASSWORD, **DB_HOST**

## Часть B. Замена URL (localhost → домен хостинга)

### Почему это нужно

В базе данных «зашит» старый адрес:

```
http://localhost/название-вашей-папки
```

Сайт на хостинге должен знать новый адрес, например:

```
http://mysite.infinityfreeapp.com
```

или `https://...` если хостинг выдал SSL.

### Почему нельзя просто заменить текст в SQL-файле

WordPress хранит часть данных в специальном формате (сериализация). Грубая замена текста в файле `.sql` может **сломать** сайт. Поэтому используем безопасные методы ниже.

---

## Метод для новичков (рекомендуем)

### Шаг 1. Временные константы в wp-config.php

Откройте `wp-config.php` на сервере. **Перед** строкой:

```php
/* That's all, stop editing! Happy publishing. */
```

добавьте (подставьте **ваш** домен хостинга):

```php
define( 'WP_HOME', 'http://ваш-домен-хостинга.com' );
define( 'WP_SITEURL', 'http://ваш-домен-хостинга.com' );
```

Пример:

```php
define( 'WP_HOME', 'http://mysite.infinityfreeapp.com' );
define( 'WP_SITEURL', 'http://mysite.infinityfreeapp.com' );
```

Сохраните. Откройте сайт — он должен открываться по новому адресу (возможно без стилей или с редиректами — это нормально на этом этапе).

### Шаг 2. Войти в админку

```
http://ваш-домен-хостинга.com/wp-admin/
```

Логин и пароль — **те же**, что задавали при локальной установке.

### Шаг 3. Плагин Better Search Replace

1. **Плагины** → **Добавить новый**
2. Найдите **Better Search Replace**
3. **Установить** → **Активировать**
4. В меню: **Инструменты** → **Better Search Replace**

Заполните:

| Поле | Значение |
|------|----------|
| Search for | `http://localhost/название-вашей-папки` |
| Replace with | `http://ваш-домен-хостинга.com` |

Включите **Run as dry run?** сначала **нет** (или сначала пробный прогон — по желанию).

Выберите все таблицы → **Run Search/Replace**.

### Шаг 4. Удалить временные строки

Вернитесь в `wp-config.php` и **удалите** строки `WP_HOME` и `WP_SITEURL`. Сохраните.

Теперь адреса хранятся в базе — дублировать в конфиге не нужно.

### Шаг 5. Постоянные ссылки

**Настройки** → **Постоянные ссылки** → **Сохранить изменения** (ничего не меняя — просто нажать Save).

Это обновит `.htaccess` и уберёт 404 на страницах.

---

## Если хостинг с HTTPS

Когда сайт открывается по `https://`:

- в шаге 3 заменяйте на `https://ваш-домен...`
- в **Настройки** → **Общие** проверьте, что оба поля начинаются с `https://`

---

## Частые ошибки

| Симптом | Что проверить |
|---------|----------------|
| Редирект на localhost | Не сделали Search Replace или неверный старый URL |
| Битые картинки | В Search Replace не хватило замены — запустите ещё раз |
| Не входит в админку | WP_HOME / WP_SITEURL временно верните, войдите, повторите замену |
