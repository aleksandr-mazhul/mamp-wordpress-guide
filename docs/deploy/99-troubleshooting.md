# 99. Решение проблем при переносе

[← Оглавление части 2](README.md)

Ошибки при переносе — нормальная часть процесса. Найдите свой симптом ниже.

---

## Error establishing a database connection

**Симптом:** «Ошибка установки соединения с базой данных».

**Проверьте в `wp-config.php` на сервере:**

| Поле | Частая ошибка |
|------|----------------|
| DB_NAME | Не то имя — скопируйте **полное** имя из панели |
| DB_USER | Не тот пользователь |
| DB_PASSWORD | Опечатка в пароле |
| DB_HOST | Поставили `localhost` по привычке с MAMP — **возьмите из панели хостинга** |

После правки сохраните файл и обновите страницу.

---

## Сайт перенаправляет на localhost

**Симптом:** при открытии домена хостинга браузер уходит на `http://localhost/...`.

**Решение:**

1. Временно добавьте в `wp-config.php`:

   ```php
   define( 'WP_HOME', 'http://ваш-домен-хостинга.com' );
   define( 'WP_SITEURL', 'http://ваш-домен-хостинга.com' );
   ```

2. Войдите в админку
3. **Better Search Replace** — замените старый localhost URL на новый (см. [раздел 08](08-wp-config-and-urls.md))
4. Удалите `WP_HOME` и `WP_SITEURL` из wp-config

---

## 404 на всех страницах кроме главной

**Симптом:** главная открывается, статьи — «Страница не найдена».

**Решение:**

1. Админка → **Настройки** → **Постоянные ссылки** → **Сохранить**
2. Проверьте, что в корне сайта есть файл `.htaccess`
3. На Apache-хостинге `.htaccess` должен содержать правила WordPress (часто создаётся при сохранении постоянных ссылок)

---

## Битые картинки

**Симптом:** картинки не грузятся, «битая» иконка.

**Причины:**

- Не все URL заменены в базе — повторите **Better Search Replace**
- Папка `wp-content/uploads` не загрузилась на сервер — проверьте по FTP/File Manager

---

## Импорт SQL: файл слишком большой

**Симптом:** phpMyAdmin отказывает в импорте из-за размера.

**Решения:**

1. Сжать `.sql` в `.zip` и импортировать zip
2. Написать в поддержку хостинга — попросить увеличить лимит
3. Для учебного сайта — удалите лишние плагины/медиа локально и экспортируйте заново

---

## Белый экран

**Решение:**

1. В `wp-config.php` временно:

   ```php
   define( 'WP_DEBUG', true );
   define( 'WP_DEBUG_DISPLAY', true );
   ```

2. Обновите страницу — появится текст ошибки
3. Часто виноват плагин — переименуйте папку плагина в `wp-content/plugins/` через File Manager
4. Смотрите логи ошибок в панели хостинга (Error Log)

---

## 500 Internal Server Error

**Возможные причины:**

- Повреждённый `.htaccess` — переименуйте в `.htaccess.bak`, зайдите в админку → Постоянные ссылки → Сохранить
- Неверные права на файлы — для папок `755`, для файлов `644` (если хостинг даёт такую настройку)
- Версия PHP слишком старая — в панели выберите PHP 8.0+

---

## Mixed content (HTTP / HTTPS)

**Симптом:** замочек в браузере перечёркнут, часть ресурсов не грузится.

**Решение:**

1. В **Настройки** → **Общие** оба URL с `https://`
2. Better Search Replace: `http://ваш-домен` → `https://ваш-домен`

---

## Не могу подключиться по FTP (FileZilla)

| Проблема | Решение |
|----------|---------|
| Connection timeout | Проверьте хост, порт, файрвол |
| Login incorrect | Логин/пароль из панели FTP, не от панели хостинга |
| Пассивный режим | FileZilla: Правка → Настройки → FTP → Passive |

---

## Чеклист перед обращением за помощью

- [ ] Файлы в правильной папке (`public_html`, не вложенная лишняя папка)
- [ ] `wp-config.php` — все 4 поля БД из панели хостинга
- [ ] SQL импортирован без ошибок
- [ ] URL заменён с localhost на домен
- [ ] Постоянные ссылки сохранены
