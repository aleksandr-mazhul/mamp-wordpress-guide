# 02. Выбор хостинга

[← Подготовка](01-prerequisites.md) | [Оглавление](README.md) | [Далее: Бэкап →](03-backup-local.md)

Хостинг — это компьютер в интернете, на котором будет жить ваш сайт. Для учебного проекта подойдёт **бесплатный** хостинг с PHP и MySQL.

---

## Минимальные требования для WordPress

| Требование | Зачем |
|------------|-------|
| PHP 8.0 или новее | WordPress написан на PHP |
| MySQL или MariaDB | База данных |
| phpMyAdmin (или аналог) | Импорт файла `.sql` |
| FTP или File Manager | Загрузка файлов сайта |
| HTTPS (желательно) | Безопасное соединение |

Если хостинг это поддерживает — подойдёт. Конкретный бренд не важен: алгоритм переноса **одинаковый**.

---

## Бесплатные варианты для учёбы

| Хостинг | Особенности |
|---------|-------------|
| [InfinityFree](https://www.infinityfree.com/) | Бесплатно, PHP + MySQL, субдомен `.infinityfreeapp.com` или свой домен |
| [AwardSpace](https://www.awardspace.com/) | Free tier, ограничения по ресурсам |

> **Честно:** бесплатный хостинг — для **обучения и тестов**, не для серьёзного бизнес-сайта. Сайт может работать медленнее, чем на платном тарифе.

После регистрации вы получите:

- панель управления (сайт хостинга → Login)
- адрес вашего сайта (домен или субдомен)
- доступ к MySQL и FTP

---

## Где что искать в панели хостинга

Названия отличаются, но смысл один:

| Ищите в панели | Зачем |
|----------------|-------|
| MySQL Databases / Базы данных | Создать базу для WordPress |
| FTP Accounts / FTP-аккаунты | Данные для FileZilla |
| File Manager / Файловый менеджер | Загрузить файлы без FTP |
| phpMyAdmin | Импорт `.sql` |

## Почему гайд универсальный

Мы **не привязываемся** к одному хостингу: кнопки могут называться иначе, но порядок действий везде похож:

1. Создать базу
2. Загрузить файлы
3. Импортировать SQL
4. Настроить `wp-config.php`
