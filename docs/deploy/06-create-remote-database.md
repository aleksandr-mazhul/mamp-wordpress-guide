# 06. Создание базы данных на хостинге

[← Загрузка файлов](05-upload-files.md) | [Оглавление](README.md) | [Далее: Импорт SQL →](07-import-database.md)

На хостинге нужна **новая пустая** база данных. Локальная база на Mac сюда не переносится автоматически — мы создадим место для импорта SQL.

---

## Шаг 1. Открыть раздел баз данных

В панели хостинга найдите:

- **MySQL Databases**
- **Базы данных**
- **MySQL Database Wizard**

## Шаг 2. Создать базу

1. Введите имя базы (латиница, без пробелов), например `my_wp_db`
2. Хостинг может добавить префикс к имени — это нормально. **Запишите полное имя**, как показывает панель

---

## Шаг 3. Создать пользователя БД

1. Имя пользователя (например `my_wp_user`)
2. Надёжный пароль — **запишите**
3. Подтвердите создание

---

## Шаг 4. Привязать пользователя к базе

1. Выберите пользователя и базу
2. Выдайте **все привилегии** (ALL PRIVILEGES / «Все права»)
3. Сохраните

---

## Шаг 5. Записать данные для wp-config

| Параметр | Ваше значение | Куда попадёт |
|----------|----------------|--------------|
| **DB_NAME** | полное имя базы из панели | `wp-config.php` |
| **DB_USER** | имя пользователя БД | `wp-config.php` |
| **DB_PASSWORD** | пароль пользователя БД | `wp-config.php` |
| **DB_HOST** | см. панель хостинга | `wp-config.php` |

### Про DB_HOST — критично!

На Mac в MAMP было `localhost`. **На хостинге часто другое значение:**

- `localhost` — бывает, но не всегда
- `sql123.hosting.com` — пример удалённого хоста MySQL
- `127.0.0.1:3306` — реже

**Копируйте DB_HOST из панели хостинга** — раздел «MySQL» / «Database details». Не угадывайте.

> **Частая ошибка:** поставить `localhost` по привычке с MAMP — сайт выдаст «Ошибка соединения с базой данных». См. [troubleshooting](99-troubleshooting.md).

База создана, но **пустая**. Следующий шаг — импорт файла `.sql` с Mac.
